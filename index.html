Apa Blockchain?

Blockchain adalah teknologi distributed ledger yang mencatat transaksi secara aman, transparan, dan tidak dapat diubah. Setiap block berisi data, timestamp, dan hash kriptografi yang menghubungkannya dengan block sebelumnya, membentuk rantai (chain) yang secure.

  • ✓ Tidak ada single point of failure
  • ✓ Data immutable dan transparent

Hash Generator

Fungsi

Mengkonversi text input menjadi hash SHA-256 yang unik dan fixed-length (64 karakter hexadecimal).

Cara Menggunakan

Ketik teks apapun di textarea di bawah. Hash akan otomatis ter-generate secara real-time tanpa perlu klik tombol.

Algoritma

Menggunakan Web Crypto API dengan fungsi crypto.subtle.digest('SHA-256', data) untuk hashing kriptografi yang aman. Setiap perubahan sekecil apapun pada input akan menghasilkan hash yang completely berbeda (Avalanche Effect).

Single Block

Fungsi

Simulasi mining satu block dengan Proof of Work untuk menemukan nonce yang menghasilkan hash valid.

Cara Menggunakan

1. Input block number dan data
2. Klik tombol "Mine Block"
3. Sistem akan mencari nonce hingga hash dimulai dengan "0000"

Algoritma

Loop iteratif yang menggabungkan block_number + data + nonce + timestamp, lalu di-hash dengan SHA-256. Nonce increment (0, 1, 2, ...) sampai hash memenuhi difficulty (4 leading zeros). Rata-rata butuh 16^4 = 65,536 iterasi.

Blockchain

Fungsi

Menampilkan chain lengkap dengan multiple blocks yang saling terhubung melalui previous hash.

Cara Menggunakan

1. Mine setiap block secara berurutan
2. Ubah data di salah satu block tengah
3. Lihat block tersebut dan semua block setelahnya menjadi invalid (border merah)

Algoritma

Validasi chain dengan memeriksa: (1) Hash setiap block valid (dimulai 0000), (2) Previous hash cocok dengan hash block sebelumnya. Jika block[i] berubah, hash-nya berubah → previous_hash block[i+1] tidak match → chain broken. Ini membuktikan immutability blockchain.

Verifikasi ECC Digital Signature

Fungsi

Mengimplementasikan digital signature menggunakan Elliptic Curve Cryptography untuk autentikasi.

Cara Menggunakan

1. Klik "Generate Key Pair" untuk membuat private dan public key
2. Ketik pesan, klik "Sign" untuk membuat signature
3. Klik "Verify" untuk membuktikan signature valid

Algoritma

Menggunakan kurva secp256k1 (sama dengan Bitcoin). Private key (256-bit random) → Public key (point pada kurva). Signing: hash message lalu enkripsi dengan private key. Verification: dekripsi signature dengan public key dan bandingkan hash.

Public Key dan Private Key dapat diedit untuk percobaan.

Simulasi Konsensus Blockchain

Fungsi

Simulasi mekanisme konsensus blockchain dengan 3 user independen (A, B, C). Lihat bagaimana node mencapai kesepakatan tentang keadaan blockchain yang valid melalui majority vote.

Cara Menggunakan

1. Kirim transaksi dari user A/B/C (pilih penerima dan jumlah)
2. Klik "Mine Semua Transaksi" untuk menambahkan ke blockchain masing-masing user
3. Edit data di salah satu chain (simulasi corrupted node)
4. Klik "Verify" untuk melihat block mana yang tidak valid (hash tidak cocok)
5. Klik "Consensus" untuk sinkronisasi menggunakan majority vote

Algoritma

Majority Vote Consensus: Setiap posisi block dibandingkan across all chains. Data yang muncul paling sering (≥2 dari 3 user) diadopsi oleh semua user. Ini mensimulasi Byzantine Fault Tolerance di mana jaringan dapat bertahan dari node yang corrupted. Mining menggunakan Proof of Work dengan difficulty "000" (3 leading zeros).
